# Test Inventory тАФ Inspector Segments Refactor

This file enumerates every test case the upfront suite (Phase 0) will land,
along with the phase that flips its `xfail` marker to passing. It is the
TDD-specific artifact that gates the rest of the refactor.

**Convention:**
- тЬЕ = passes immediately on Phase 0 commit (asserts current behavior that survives the refactor).
- ЁЯФБ N = xfail-marked with `reason="phase-N"`; flips green when Phase N commits.
- ЁЯЖХ = new fixture file, not a test case.

A test never moves between phases without a corresponding plan amendment.

## Maintenance protocol

- **A test never moves between phases without a plan amendment.** If during implementation an agent decides a test "really" should flip in a later phase, that's a plan deviation тАФ fire S4 (out-of-scope edit) or amend the plan in a small commit before the phase commits.
- **No new tests added mid-phase to make a failing test pass.** Add the test in Phase 0, mark xfail, fix in the appropriate phase. A new test added during Phase N that asserts current behavior is fine; a new test added that asserts behavior the phase implements is suspect.
- **Snapshot regeneration**: when classifier behavior intentionally changes (Phase 2 unification), run `python -m inspector.tests.parity.snapshot_expected_outputs` to regenerate `expected/*.classify.json`. The regeneration commit is part of Phase 2's diff.
