Download Format

Each release provides per-reciter zip files containing alignment data. Here's what's inside:

Zip Structure

reciter_name.zip
  info.json          Reciter metadata (name, riwayah, source)
  audio.json         Audio URL index (surah/ayah to URL mapping)
  segments.json      Verse-level alignment segments
  timestamps.json    Word-level timestamps (MFA)
  timestamps_full.json  Word + letter + phoneme timestamps (if available)

Key Files

  • segments.json — Flat dict keyed by "surah:ayah". Each value is an array of [word_from, word_to, time_from_ms, time_to_ms] segments. First entry is "_meta" with pipeline settings.
  • timestamps.json — Flat dict keyed by "surah:ayah". Each value is an array of [word_idx, start_ms, end_ms] word timestamps.
  • timestamps_full.json — Same keys. Each value has "words": [[word_idx, start_ms, end_ms, [[char, start_ms, end_ms], ...]], ...]

Shared Reference Files

Each release also includes surah_info.json (114 surahs with verse/word counts) and qpc_hafs.json (Quran text reference). All timestamps are in milliseconds.
